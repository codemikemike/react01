## 🌐 Overview

React01 is a modern, full-stack web application that demonstrates best practices in web development. Built with React 18 and Express.js, it features a beautiful dark theme, smooth animations, and a fully functional backend with SQLite database integration.

Perfect for:
- 🎓 Learning full-stack development
- 🏗️ Starting a new project with a solid foundation
- 📚 Understanding React Router and Express API patterns
- 🎨 Exploring modern UI/UX design principles

## 📥 Installation

### Prerequisites

Before you begin, ensure you have the following installed:
- **Node.js** (v14 or higher) - [Download](https://nodejs.org/)
- **npm** (comes with Node.js) or **yarn**
- **Git** - [Download](https://git-scm.com/)

### Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/codemikemike/react01.git
cd react01

# 2. Install dependencies
npm install

# 3. Start the application (both frontend & backend)
npm run dev
```

That's it! The application will open automatically in your browser.

- 🌐 **Frontend**: [http://localhost:3000](http://localhost:3000)
- 🔌 **Backend API**: [http://localhost:5001](http://localhost:5001)

### Alternative Installation Methods

<details>
<summary><b>Run Frontend and Backend Separately</b></summary>

```bash
# Terminal 1 - Start Backend (Port 5001)
npm run server

# Terminal 2 - Start Frontend (Port 3000)
npm start
```

</details>

<details>
<summary><b>Production Build</b></summary>

```bash
# Create optimized production build
npm run build

# The build folder is ready to be deployed
# Serve with a static server
npx serve -s build
```

</details>

## 🔌 API Documentation

### Base URL
```
http://localhost:5001/api
```

### Endpoints Overview

| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|---------------|
| `GET` | `/api/health` | Health check | ❌ |
| `POST` | `/api/contact` | Submit contact form | ❌ |
| `GET` | `/api/contacts` | Get all contacts | ❌ |
| `GET` | `/api/contacts/:id` | Get single contact | ❌ |
| `PATCH` | `/api/contacts/:id/status` | Update contact status | ❌ |
| `DELETE` | `/api/contacts/:id` | Delete contact | ❌ |
| `GET` | `/api/stats` | Get statistics | ❌ |

### Detailed Endpoints

#### 1️⃣ Health Check

```http
GET /api/health
```

**Response:**
```json
{
  "status": "ok",
  "message": "Server is running"
}
```

---

#### 2️⃣ Submit Contact Form

```http
POST /api/contact
Content-Type: application/json
```

**Request Body:**
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "subject": "Project Inquiry",
  "message": "I'm interested in collaborating on a project."
}
```

**Success Response (200):**
```json
{
  "success": true,
  "message": "Contact message received successfully",
  "id": 1
}
```

**Error Response (400):**
```json
{
  "success": false,
  "error": "All fields are required"
}
```

**Validation:**
- All fields are required
- Email must be valid format
- Server-side validation with regex

---

#### 3️⃣ Get All Contacts

```http
GET /api/contacts
```

**Response:**
```json
{
  "success": true,
  "count": 5,
  "contacts": [
    {
      "id": 1,
      "name": "John Doe",
      "email": "john@example.com",
      "subject": "Project Inquiry",
      "message": "I'm interested in...",
      "created_at": "2026-04-07T10:00:00.000Z",
      "status": "unread"
    }
  ]
}
```

---

#### 4️⃣ Get Contact by ID

```http
GET /api/contacts/:id
```

**Example:** `GET /api/contacts/1`

**Success Response (200):**
```json
{
  "success": true,
  "contact": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "subject": "Project Inquiry",
    "message": "I'm interested in...",
    "created_at": "2026-04-07T10:00:00.000Z",
    "status": "unread"
  }
}
```

**Error Response (404):**
```json
{
  "success": false,
  "error": "Contact not found"
}
```

---

#### 5️⃣ Update Contact Status

```http
PATCH /api/contacts/:id/status
Content-Type: application/json
```

**Request Body:**
```json
{
  "status": "read"
}
```

**Valid Status Values:**
- `unread` - New submission
- `read` - Viewed by admin
- `responded` - Response sent

**Success Response (200):**
```json
{
  "success": true,
  "message": "Contact status updated"
}
```

---

#### 6️⃣ Delete Contact

```http
DELETE /api/contacts/:id
```

**Example:** `DELETE /api/contacts/1`

**Success Response (200):**
```json
{
  "success": true,
  "message": "Contact deleted successfully"
}
```

---

#### 7️⃣ Get Statistics

```http
GET /api/stats
```

**Response:**
```json
{
  "success": true,
  "stats": {
    "total": 10,
    "unread": 3,
    "read": 5,
    "responded": 2
  }
}
```

### cURL Examples

```bash
# Submit a contact form
curl -X POST http://localhost:5001/api/contact \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Smith",
    "email": "jane@example.com",
    "subject": "Question",
    "message": "How do I get started?"
  }'

# Get all contacts
curl http://localhost:5001/api/contacts

# Update contact status
curl -X PATCH http://localhost:5001/api/contacts/1/status \
  -H "Content-Type: application/json" \
  -d '{"status": "read"}'

# Delete a contact
curl -X DELETE http://localhost:5001/api/contacts/1

# Get statistics
curl http://localhost:5001/api/stats
```

### Database Schema

```sql
CREATE TABLE contacts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  subject TEXT NOT NULL,
  message TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  status TEXT DEFAULT 'unread'
);
```

## 🔧 Development

### Development Scripts

```json
{
  "start": "react-scripts start",           // Frontend dev server (port 3000)
  "server": "nodemon server/server.js",     // Backend server (port 5001)
  "dev": "concurrently \"npm run server\" \"npm start\"",  // Both together
  "build": "react-scripts build",           // Production build
  "test": "react-scripts test",             // Run tests
  "eject": "react-scripts eject"            // Eject from CRA (irreversible)
}
```

### Environment Variables

Create a `.env` file in the root directory (optional):

```env
# Frontend (Create React App)
PORT=3000
BROWSER=none                    # Don't auto-open browser

# Backend
SERVER_PORT=5001
NODE_ENV=development
DATABASE_PATH=./server/contacts.db
```

### Database Management

```bash
# View database contents
sqlite3 server/contacts.db "SELECT * FROM contacts;"

# Reset database
rm server/contacts.db
npm run server  # Will auto-recreate with schema

# Backup database
cp server/contacts.db server/contacts.backup.db
```

### Adding New Features

<details>

```javascript
import NewPage from './pages/NewPage';

<Route path="/new" element={<NewPage />} />
```

3. Add navigation link in `src/components/Navbar.js`:
```javascript
<Link to="/new" className="navbar-link">New Page</Link>
```

</details>

<details>
<summary><b>Adding a New API Endpoint</b></summary>

```javascript
});
```

## 🐛 Troubleshooting

<details>
<summary><b>Port Already in Use</b></summary>

```bash
# Kill process on port 3000
lsof -ti:3000 | xargs kill -9

# Kill process on port 5001
lsof -ti:5001 | xargs kill -9
```

</details>

<details>
<summary><b>Database Locked Error</b></summary>

```bash
# Close all connections and restart
rm server/contacts.db
npm run server
```

</details>

<details>
<summary><b>Module Not Found</b></summary>

```bash
# Reinstall dependencies
rm -rf node_modules package-lock.json
npm install
```

</details>

<details>
<summary><b>CORS Issues</b></summary>

Ensure `package.json` has the proxy setting:
```json
"proxy": "http://localhost:5001"
```

</details>
